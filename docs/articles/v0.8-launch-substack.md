## The week the vocabulary caught up

In early April 2026, a few different threads converged. Akshay Pachaar
published *"The Anatomy of an Agent Harness"* — a synthesis of how
Anthropic, OpenAI, LangChain, and Perplexity actually structure
production agents. LangChain's Vivek Trivedy formalized the canonical
distinction: *"If you're not the model, you're the harness."* A
research project hit a 76.4% pass rate on TerminalBench by having an
LLM optimize the harness itself, beating hand-designed baselines.

The term was new. The work wasn't.

I had been quietly shipping the same thing for months. Not because I
read the paper — because the model kept forgetting things, and every
fix I built clustered around the same architectural pattern. By the
time the vocabulary arrived, I had nine days of git history showing me
building an agent harness.

This is the v0.9.2 launch post for **Mnemosyne** — a local-first
agent substrate that, as of today, ships every one of the twelve
components Akshay's article identifies as production-grade harness
machinery. Most are ✓. Three are partial, with concrete designs for
v0.10. The audit is a runnable doc:
[`docs/HARNESS.md`](https://github.com/atxgreene/Mnemosyne/blob/main/docs/HARNESS.md).

Six weeks after I started this project, Rohit Yadav published a
55-directory, 331-module teardown of Claude Code's production
architecture. Half of what he documents is what we ship. Half is
v0.10 roadmap. None of it is handwaved.

## What I actually built

![Mnemosyne system architecture: Channels → Brain → Tools + 6-tier ICMS memory (L0 instinct through L5 identity) + Inner Dialogue + Dream Consolidation + Meta-Harness self-improvement loop](../images/architecture-overview.png)

Mnemosyne is a Python package — `pip install mnemosyne-harness` —
that gives a local LLM the things context windows can't carry
between sessions: persistent memory, stable identity across model
swaps, learned user-pattern intuition, observable self-regulation,
and a closed loop for self-improvement. It is **stdlib-only.** Zero
runtime dependencies. The whole substrate is auditable in an
afternoon. 25 console scripts land on your `$PATH` after install;
your data lives in `~/projects/mnemosyne/` as plain SQLite, JSONL,
and Markdown files. Delete the framework tomorrow and your
knowledge survives.

The core components, with the dates I shipped them:

- **April 9.** [`docs/ARCHITECTURE.md`](https://github.com/atxgreene/Mnemosyne/blob/main/docs/ARCHITECTURE.md):
  the four-layer stack (deployment / meta-harness / base harness /
  engine), with a thesis literally titled *"The harness matters more
  than the model."* Six weeks before Akshay's article gave the pattern
  a name.
- **April 14.** v1 agent framework — brain + memory + models + skills.
  The `Backend(provider="...")` interface that abstracts 19 LLM
  providers behind one call, because the model is supposed to be
  swappable.
- **April 15.** Identity lock + 40-prompt jailbreak suite. The agent
  is named Mnemosyne. It will say it's Mnemosyne whether the
  underlying model is Claude, Qwen, or GPT-4. Slip rate is measured.
- **April 15.** Triage → proposer → apply: the agent observes its own
  failures, drafts proposals to fix them, applies them with
  measurement. CREAO-inspired closed loop, no external orchestrator.
- **April 15.** Inner dialogue — Planner / Critic / Doer / Evaluator
  phases that run inside one Brain, opt-in via config.
- **April 16.** v0.6: self-calibration. Predictions become first-class
  telemetry events. The agent emits a confidence number; the system
  observes the outcome; calibration becomes a measurable trait.
- **April 16.** v0.7: 5-tier ICMS memory with ACT-R-inspired decay
  and Hebbian reinforcement. L1 hot, L2 warm, L3 cold, L4 pattern
  (clusters from a compactor), L5 identity (human-approved core
  values). Continuity Score benchmark — 50 scenarios, 6 categories,
  cross-session subset.
- **April 16.** v0.7.1: substrate AND→OR recall fallback. The
  Continuity dryrun jumped from 0.34 to 0.96 — not by adding LLM
  cleverness, but by fixing how FTS5 falls back when strict
  conjunction misses.
- **April 16.** v0.8.0: **Instinct overlay**. User-pattern signals
  distilled offline into L4 rows that the Brain consults on every
  turn before query-relevance retrieval. The compactor gets an
  audit pass. Two real bottlenecks (per-row UPDATE in `search()` and
  `apply_decay()`) collapsed into batched `executemany` calls. A
  benchmark skeleton (`bench/locomo.py`) lands so head-to-heads
  against Mem0 are runnable, not just describable.
- **April 16.** v0.9.0: **Instinct promoted to its own tier (L0)**.
  Six tiers now: L0 instinct / L1 hot / L2 warm / L3 cold / L4
  pattern / L5 identity. L5 Identity gains the documented
  "Reflection" role — the distillation pass that writes L0
  user_instinct rows from L5 + lower is the Reflection → Instinct
  loop that was informal in v0.8 and is a first-class substrate
  primitive in v0.9.
- **April 17.** v0.9.2 (today): **tool result budgeting**. A single
  `cat` on a 1 MB log file no longer fills the context window with
  noise; tool outputs over the configured cap persist to disk and
  the model sees a preview + file reference. Defends against a
  specific production failure mode that Claude Code's 45+ tool
  subsystem exists to prevent.

Ten days. Eleven versions. One unbroken thread: **the harness is
the product**.

## What the field caught up to

The harness pattern Akshay describes has twelve components. I'll
walk through three of them where Mnemosyne went further than the
baseline, because the rest of the table is in
[`docs/HARNESS.md`](https://github.com/atxgreene/Mnemosyne/blob/main/docs/HARNESS.md).

**Memory.** Most production harnesses use a vector DB — Pinecone,
Qdrant, Chroma — plus an embedding pipeline plus optional graph
storage (Neo4j, Memgraph). It works. It also pulls in three vendors,
costs money to query, and degrades into "97% junk in production
audits" if you don't tune the eviction rules carefully (a number Mem0
themselves report).

Mnemosyne's memory is **one SQLite database with FTS5.** Five tier
constants on a single `memories` table. ACT-R base-level activation
math drives decay, multiplied per kind so identity-class rows
(`core_value`, `preference`) decay at 0.1× and operational rows
(`failure_note`, `tool_result`) decay at 3×. Reinforce on read with
asymptotic Hebbian (`s + 0.05 * (1 - s)` — pushes toward 1.0, never
exceeds it). At 5,000 rows: writes 0.13 ms each, search 2.4 ms,
full decay scan 100 ms. At 50,000: decay scan 6.15 s. Zero
dependencies. Zero cloud. Zero junk-accumulation crisis, because the
substrate itself prevents it.

The 6-tier hierarchy with L0 Instinct makes user-pattern signals show
up *before* retrieval runs. If you've told the agent three times that
you prefer terse answers, by the next session it's in a fast-path
system block that injects on every turn — not because the model
"remembered," but because the substrate distilled the pattern offline
and put it in front of the next inference call. That's the
Reflection → Instinct loop: slow deliberate consolidation gradually
shapes fast automatic reaction. Procedural memory in cognitive
science; L0 in code. Fifty lines of SQL, zero new dependencies.

**Identity.** Most agents lose their persona the moment you swap
models. Try it: tell ChatGPT it's "Aria, a helpful music assistant,"
and watch how fast it forgets in a long conversation, or the moment
you switch from GPT-4 to Claude. Mnemosyne ships a 4-layer identity
lock: a fixed system-prompt preamble, a regex post-filter, an
`IDENTITY.md` user extension, and (as of v0.7) an L5 identity-memory
tier that re-injects core values into every turn's system prompt
independent of retrieval. Tested against a 40-prompt jailbreak suite
in `scenarios/jailbreak.jsonl`. The slip-rate metric is reproducible.
The identity persists across all 19 supported backends.

**Self-calibration.** The cognitive-OS checklist at
[`docs/COGNITIVE_OS.md`](https://github.com/atxgreene/Mnemosyne/blob/main/docs/COGNITIVE_OS.md)
lists five operational properties of a "cognitive operating system."
The fourth is self-calibration: the runtime emits predictions as
first-class events, observes outcomes, and the calibration score
itself becomes a measurable agent trait. Most systems don't have
this — they call a model, accept the output, move on. Mnemosyne's
v0.6 release wired predictions into the telemetry layer with a
horizon-bounded scoring loop and a triage rule
(`prediction_overconfident`) that fires when the agent claims high
confidence and is wrong. Calibration shows up as the 17th trait on
the avatar dashboard.

The whole checklist — all five rows — is green as of v0.7. Each row
has a verify command a user can run. That's the gate; until all five
were green, the README tagline read "cognitive substrate." Now it
honestly reads "cognitive OS."

## Honest about what's not done

Three of the twelve harness components are partial (see HARNESS.md
for the table):

- **Context management** lacks observation masking on long tool-use
  loops and intra-turn compaction. JetBrains' Junie hides old tool
  outputs while keeping tool calls visible; we don't. v0.9 target.
- **Error handling** is ad-hoc retries plus exception catches. It
  works, but it doesn't categorize errors into LangGraph's clean
  4-bucket schema (transient / LLM-recoverable / user-fixable /
  unexpected). v0.9 target.
- **Verification loops** are post-hoc only — triage clusters errors
  after the fact. Per-turn verification (rules-based, visual,
  LLM-as-judge) à la Boris Cherny's *"give the model a way to verify
  its work, quality goes up 2-3×"* is not yet wired. v0.9 target.

Also: **we have not run LOCOMO numbers against Mem0 yet.** The
benchmark runner ships in v0.8.0 (`bench/locomo.py`) with adapters
for both substrates, but the head-to-head comparison requires running
the script with an LLM endpoint and judge model. We have the
plumbing. We don't yet have the numbers. The article you're reading
will get a follow-up the moment we do.

What we do have are reproducible substrate numbers from
`mnemosyne-continuity dryrun`:

| Category | v0.7.0 | v0.7.1 | Δ |
|---|---:|---:|---:|
| preference | 0.42 | 1.00 | +0.58 |
| fact | 0.50 | 1.00 | +0.50 |
| project | 0.17 | 0.92 | +0.75 |
| decision | 0.00 | 1.00 | +1.00 |
| rule | 0.50 | 0.83 | +0.33 |
| **aggregate** | **0.34** | **0.96** | **+0.62** |

The dryrun is the substrate-only lower bound — it measures what the
memory layer alone can resolve, with no LLM in the loop. Live-model
upper bounds (Continuity Score with GPT-4 or a local Qwen model
reasoning over retrieved context) are what users measure on their
own hardware.

## Why this matters now

As "agent harness" becomes a recognized discipline, two things are
happening simultaneously: (1) production teams are realizing the
infrastructure around the model is where the engineering lives, and
(2) the local-first / offline / auditable / no-vendor-lock-in
quadrant is wide open because most production frameworks default
to cloud-managed everything.

Mnemosyne sits in that quadrant. By design. We don't have a managed
service and we're not going to build one. We don't collect telemetry
and we can't disable your install. The substrate is what survives
when AI tooling consolidation eats the rest of the field, because
your `~/projects/mnemosyne/` directory is plain files you own.

If you're building a personal agent that should remember you across
sessions, hold an identity across model swaps, observe its own
failure modes, and improve over time — and you don't want to ship
your conversation history to a third party to make any of that
work — Mnemosyne is the substrate.

## Try it tonight

```sh
pip install mnemosyne-harness
mnemosyne-wizard.sh         # one-time credential setup
mnemosyne-serve &           # daemon + dashboard on :8484
open http://127.0.0.1:8484/ui
```

The owl avatar updates in real time as the agent uses memory. Walk
through `docs/QUICKSTART.md` for the 10-line first conversation. Walk
through `docs/HARNESS.md` if you want to see the 12-component audit
with verify commands.

If you have a local model running in LM Studio or Ollama and want
to run the Continuity Score benchmark against it:

```sh
mnemosyne-continuity run \
    --scenarios scenarios/continuity.jsonl \
    --model qwen2.5:7b --provider ollama \
    --out /tmp/report.json
```

Report the numbers in an issue — we'd genuinely like to see them.
