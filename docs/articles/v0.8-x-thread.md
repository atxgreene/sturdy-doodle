## Thread (1/12 → 12/12)

**1/** Most agent frameworks are <6 months old.

I started shipping one in early April. By the time "agent harness"
became a recognized term mid-month, I had built all 12 components
the field is now writing about — without the vocabulary.

Here's what we got right by accident, and what's still ahead. 🧵

**2/** The term got formalized when @akshay_pachaar published "The
Anatomy of an Agent Harness" — synthesizing how Anthropic, OpenAI,
LangChain, and Perplexity actually wrap their models in production.

12 components. We had 9 ✓ and 3 partial. Self-audit:
github.com/atxgreene/sturdy-doodle/blob/main/docs/HARNESS.md

**3/** The substrate is called Mnemosyne. `pip install
mnemosyne-harness`. Stdlib-only. 25 console scripts. Zero runtime
dependencies. Local-first. SQLite + JSONL + Markdown — your
knowledge survives without the framework.

If the framework disappeared tomorrow your data is plain files.

**4/** Memory: 5 tiers (hot → warm → cold → pattern → identity).

Not a vector DB. Not a graph. SQLite + FTS5 with `tier`, `kind`,
`strength` columns. ACT-R-inspired decay. Hebbian reinforcement on
read. 50K rows: writes 0.16 ms, search 2.4 ms, decay scan 6.15s.

No 97%-junk-accumulation problem.

**5/** Identity: stable across 19 LLM backends.

Tell the agent it's "Mnemosyne" once. Switch from Qwen → Claude →
GPT-4 → back. Identity holds. Tested against a 40-prompt jailbreak
suite. Slip rate measured.

Most agents lose persona on model swap. This one doesn't.

**6/** Self-regulation: SVG dashboard with 17 derived agent traits.

Avatar evolves in real time. Every visual change maps to one
telemetry event you can grep. No black-box "personality engine."

mnemosyne-serve & ; open http://127.0.0.1:8484/ui

**7/** Self-calibration (v0.6): predictions are first-class telemetry.

The agent emits a confidence number. The system observes the outcome.
Calibration becomes a measurable agent trait. Triage clusters
overconfident-wrong claims separately from low-confidence drift.

**8/** Self-improvement (v0.2 onward): triage → proposer → apply,
running autonomously inside the daemon.

The agent observes its own failures, drafts proposals, applies them
with measurement. CREAO-inspired closed loop. No external orchestrator.

**9/** v0.7 added an Instinct overlay — user-pattern signals distilled
offline into a fast-path system block.

Tell the agent 3× you prefer terse responses. Next session: it's
already in the prompt before retrieval runs. Reflection→intuition
loop, modeled on procedural memory, ~50 lines of SQL.

**10/** Continuity Score benchmark, 50 scenarios, 6 categories.

Substrate-only dryrun (no LLM): 0.96 aggregate, 1.0 cross-session.

Live-model upper bound is yours to measure. Run on your local box:
`mnemosyne-continuity run --provider ollama --model qwen2.5:7b`

**11/** Cognitive-OS checklist (docs/COGNITIVE_OS.md): 5 properties,
each with a verify command. All 5 green as of v0.7.

Until they were green, the README tagline read "cognitive substrate."
Now it honestly reads "cognitive OS."

Marketing language gated by tests.
